dploy

The DC/OS deployment tool for appops allows you to create, deploy and manage services and apps composed of microservices (ÂµS):

  • It is simple: its workflow comprises 8 commands and that's that.
  • It is stateless: state is exclusively kept in (local) descriptor and spec filesâ€”a collection of Marathon app specs expressed in JSON.
  • It is self-contained: written in Go, dploy is a single binary incl. all dependencies.

The main motivation behind the creation of dploy was the desire to help out developers to strengthen their ops skills. We use convention over configuration in dploy and given the limited command set it should be fairly easy to get started. So, now would be a good time to either install dploy and start using is, or maybe you want to see it in action first?

Installation

From source:

$ go get github.com/mhausenblas/dploy
$ go build

To simplify the DC/OS oauth handling, you can create a SSH tunnel from your local machine to the DC/OS master like so:

$ ssh -i ~/.ssh/MYKEY core@MYMASTER -f -L 8080:localhost:8080 -N

From here on, you can use http://localhost:8080 for the Marathon URL (marathon_url) in the dploy.app file.

Usage

Logging

To set the log output level for dploy, use the environment variable DPLOY_LOGLEVEL. For example, to set it globally use: 

$ export DPLOY_LOGLEVEL=debug

Of course, you can also set the log output level on a per-run basis, using $ DPLOY_LOGLEVEL=info dploy dryrun. Note that the default valueâ€”if you don't set the environment variableâ€”is DPLOY_LOGLEVEL=error.

Workflow

dploy init

This command creates an app descriptor dploy.app.

Usage:

$ dploy init [-location=DIR]

With the following optional parameter:

  • If -location is not present, dploy.app is created in the current dir
  • If -location is present, you can specify its value DIR either absolute (e.g., /tmp/) or relative to the directory dploy is executed, for example, ../. means create dploy.app in the parent dir

Example:

$ dploy init

dploy dryrun

This command validates the app descriptor by checking if Marathon is reachable and also checks if the app spec directory is present, incl. at least one Marathon app spec.

Usage:

$ dploy dryrun

dploy run

This command launches the app as defined in the app descriptor and the app specs. It scans the specs/ directory for Marathon app specs and launches them using the Marathon API.

Usage:

$ dploy run

Example:
