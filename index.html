dploy

If you are in an appops role and want to run microservices in a simple and convenient way, dploy might be something for you. Essentially, dploy allows you to create, deploy and manage apps based on a collection of Marathon app specs written in JSON. To set up and manage your app across the entire lifecycle, we use convention over configuration, for example the location of resources.

Dependencies

  • DC/OS 1.7
  • github.com/gambol99/go-marathon, an API library for working with Marathon.
  • github.com/Sirupsen/logrus, a logging library.

Workflow

dploy init

dploy init creates a new app for you, that is, a dploy.app file with default values is created in -location

dploy dryrun

dploy dryrun validates deployment of the app by checking if DC/OS cluster is valid, validates app specs, etc.

dploy run

dploy run launches the app using the Marathon API
